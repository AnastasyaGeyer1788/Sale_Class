# 🛍️ Sale_Class
Проект реализует систему управления товарами и категориями для интернет-магазина электроники
с использованием объектно-ориентированного программирования на Python.

## 🏗️ Архитектура классов

### Класс Product

**Расположение:** `src/product.py`

**Назначение:** Представляет товар в магазине

**Атрибуты:**
- `name` (str) - название товара
- `description` (str) - описание товара
- `price` (Union[int, float]) - цена товара
- `quantity` (int) - количество на складе

**Конструктор:**
```python
def __init__(self, name: str, description: str, price: Union[int, float], quantity: int): 
```
### Класс Category

**Расположение:** `src/category.py`

**Назначение:** Представляет категорию товаров

**Атрибуты:**
- `name` (str) - название категории
- `description` (str) - описание категории
- `products` (list[Product]) - список товаров в категории

**Статические атрибуты:**
- `category_count` (int) - общее количество категорий
- `product_count` (int) - общее количество товаров

**Конструктор:**
```python
def __init__(self, name: str, description: str, products: list[Product]):
```
### Специализированные классы товаров

**Дополнительные свойства:**

## Особенности реализации

### Валидация типов
- При добавлении товаров в категорию проверяется, что объект является экземпляром Product или его подклассов
- При сложении товаров проверяется, что товары относятся к одной категории
- Используются функции `isinstance()` и `issubclass()` для проверки типов

### Защищенные атрибуты
- Цена товара защищена от некорректных значений
- Список товаров в категории инкапсулирован

### Счетчики
- Автоматический подсчет количества категорий и товаров
- Сброс счетчиков перед каждым тестом

## Реализованная новая функциональность
### 🎯 Абстрактный базовый класс BaseProduct
Создан абстрактный класс BaseProduct, который стал родительским для всех продуктов:
* Абстрактные методы: __init__, __str__, __add__, свойства price
* Инкапсуляция: Приватный атрибут __price перемещен в базовый класс
* Валидация: Проверка цены на отрицательные и нулевые значения

### 🔄 Класс-миксин CreationLoggerMixin
Реализован миксин для логирования создания объектов:

* Автоматическое логирование: При создании объекта выводится информация в консоль
* Форматированный вывод: Параметры создания выводятся в читаемом формате
* Метод repr: Реализовано строковое представление для воссоздания объектов

### ✨ Ключевые возможности
1. Автоматическое логирование создания всех продуктов

3. Строгая типизация при сложении продуктов
4. Обратная совместимость со старым кодом
